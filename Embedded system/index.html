EMBEDDED SYSTEMS - OVERVIEW

System:

A system is an arrangement in which all its unit assemble work together according to a set of rules. It can also be defined as a way of working, organizing or doing one or many tasks according to a fixed plan. For example, a watch is a time displaying system. Its components follow a set of rules to show time. If one of its parts fails, the watch will stop working. So we can say, in a system, all its subcomponents depend on each other.

Embedded system :

As its name suggests, Embedded means something that is attached to another thing. An embedded system can be thought of as a computer hardware system having software embedded in it. An embedded system can be an independent system or it can be a part of a large system. An embedded system is a microcontroller or microprocessor based system which is designed to perform a specific task. For example, a fire alarm is an embedded system; it will sense only smoke

components:-

  1. hardware:
    • Microcontroller/Microprocessor: The central processing unit (CPU) that runs the software
    • memory: ROM (Read-Only Memory) for storing firmware and RAM (Random Access Memory) for runtime data.
    • Input/Output Interfaces: Ports and interfaces for communication with external devices, sensors, and actuators.
    • Power Supply: of power, often optimized for low energy consumption.
  2. software:
    • firmware: Low-level software programmed into the device's ROM.
    • Operating System: Some embedded systems run a real-time operating system (RTOS), while others may operate with minimalistic firmware.
    • Application Software: High-level programs that perform specific tasks.
  3. RTOS :
    • It has Real Time Operating system (RTOS) that supervises the application software and provide mechanism to let the processor run a process as per scheduling by following a plan to control the latencies. RTOS defines the way the system works. It sets the rules during the execution of application program. A small scale embedded system may not have RTOS

Examples of Embedded Systems:-

  1. Consumer Electronics: Smartphones, MP3 players, digital cameras.
  2. automotive: Engine control units (ECUs), anti-lock braking systems (ABS), airbag control systems.
  3. Home Appliances: Microwave ovens, washing machines, thermostats.
  4. industrial: Robotics, programmable logic controllers (PLCs), industrial automation systems
  5. Medical Devices: Pacemakers, insulin pumps, diagnostic equipment.

Development Tools:-

  • Compilers and Assemblers: Convert code into machine language
  • debuggers: Tools to test and debug embedded software.
  • Integrated Development Environments (IDEs): Software applications that provide comprehensive facilities to programmers for software development.
  • Simulators and Emulators: Tools that mimic the behavior of the embedded system for testing purposes.

    • Trends in Embedded Systems:-

    1. Internet of Things (IoT): Increasing connectivity of embedded systems to the internet, allowing for remote monitoring and control.
    2. AI and Machine Learning: Integration of AI algorithms in embedded systems for smarter and more autonomous operations.
    3. Edge Computing: Processing data closer to where it is generated, reducing latency and bandwidth use.
    4. Energy Efficiency: Focus on reducing power consumption, particularly for portable and wearable devices.

    Embedded systems are a crucial part of modern technology, powering a wide range of devices and applications. Their design and development require a specialized approach to meet the unique requirements of their intended functions. As technology advances, embedded systems continue to evolve, becoming more powerful, efficient, and interconnected.

    Characteristics of Embedded Systems :

    Embedded microprocessors are similar in basic principle to those of ordinary desktop computers, but have higher working stability, lower power consumption, and strong adaptability to the environment (such as temperature, humidity, electromagnetic field, vibration, etc.) , smaller size, and more integrated functions.

    In the field of desktop computers, the main index when comparing processors is the calculation speed. From the 386 computer with a main frequency of 33MHz to the PenTIum 4 processor with a main frequency of 3GHz, the improvement of speed is the change that users are most concerned about. field, the situation is completely different. The choice of embedded processor must be based on the design requirements, among many factors such as performance, power consumption, function, size and packaging form, SoC level, cost, commercial considerations, etc., to make a compromise and choose the best.

    1. Real-Time Operation:
      • Deterministic Response: Embedded systems often need to respond to events within a strict timing constraint, making real-time performance critical.
      • Predictability: The system’s behavior should be predictable and reliable within specified time frames.
    2. Resource Constraints:
      • Limited Memory and Storage: They usually operate with limited RAM, ROM, and storagecapacity.
      • Restricted Processing Power: microcontrollers or microprocessors used are often lesspowerful compared to those in general-purpose computers.
    3. Reliability and Stability:
      • High Reliability: Embedded systems must function reliably over long periods, often in harsh or demanding environments.
      • Stability: The system should operate without failures, especially in critical applications like medical devices or automotive systems.
    4. Specific Functionality:
      • Dedicated Purpose: Designed to perform specific tasks or functions, often with optimized hardware and software.
      • Task-Specific: Unlike general-purpose computers, embedded systems are not meant for a broad range of tasks.
    5. Small Size and Lightweight:
      • Compact Design: Often designed to be small and lightweight, fitting into the physical constraints of the larger system or device they are part of.
      • Integration: Tightly integrated with the hardware it controls, often resulting in a highly optimized form factor.
    6. Low Power Consumption:
      • Energy Efficiency: Many embedded systems are designed to consume minimal power, especially important for battery-powered devices.
      • Power Management: Often includes advanced power management features to extend battery life and reduce energy consumption.
    7. Embedded Software (Firmware):
      • Firmware: The software is typically embedded into the hardware, stored in non-volatile memory like ROM or flash.
      • Low-Level Programming: written in low-level languages like C or assembly for efficient control of hardware.
    8. Interfacing and Communication:
      • I/O Interfaces: with various input/output interfaces to communicate with other devices, sensors, and actuators.
      • Communication Protocols: Use specific protocols (e.g., I2C, SPI, UART) for data exchange within the system and with external components.
    9. Environmental Constraints:
      • Operating Conditions: Designed to operate under specific environmental conditions, such as temperature ranges, humidity, and vibration.
      • Durability: Built to withstand the physical demands of their operating environment.
    10. Cost Sensitivity:
      • Cost-Effective: designed with cost constraints in mind, balancing performance and functionality with economic considerations.
      • Mass Production: Typically produced in large quantities, which can influence the design and component selection to minimize costs.
    11. Minimal User Interface:
      • Simple Interfaces: May have limited or no user interface, relying on other systems or devices for control and monitoring.
      • Automated Operation: Often operates autonomously without direct human intervention onceprogrammed.
    12. Long Lifecycle:
      • Longevity: Designed for long-term use, often without the need for frequent updates or replacements.
      • Maintenance: Requires minimal maintenance and updates compared to general-purpose computers.

    Basic Structure of an Embedded Systems :

    The basic structure of an embedded system includes a processor, sensor, memory, D-A converters, input/output (I/O) interfaces, graphical user interfaces, and software.

    Here’s an overview of these components:

    The processor is the heart of the embedded system. It is a central processing unit (CPU) that allows all the integrations of the embedded system. The design of the type of processor used in an embedded system depends on the application. For example, a simple embedded system for a toy may use a microcontroller, while a more complex embedded system, such as the one in an airplane, may use a microprocessor.

    • Sensor:
    • To make it comprehensible for an observer or any electrical instrument, the sensor converts the physical quantity after measurement- into electrical signals.
    • A sensor is a device that converts physical quantity into electrical signals. It senses the conditions of the external world and provides input to the embedded system.
    • A-D Converter:
    • The analog signal sent by the sensor is converted into a digital signal via an analog-to-digital converter.
    • Processor & ASIC:
    • It looks after processing the data and measuring and storing output in the memory.
    • D-A Converter:
    • It converts the digital data to Analog data.
    • D-A converters convert digital signals into analog signals. I/O interfaces connect the embedded system to the outside world. They provide a means for the system to receive input from and send output to the external world.
    • Actuator:
    • The permitted output is stored when an actuator compares the D-A Converter’s output to the real (anticipated) output stored in it.
    • An actuator compares the output given by the D-A Converter to the actual (expected) output stored in it and stores the approved output.
    • Memory:
    • It refers to storage components where data can be temporarily or permanently stored for various purposes in the system.
    • Memory stores data and programs. There are two types of memory: volatile and non-volatile. Volatile memory, such as RAM, loses its contents when the power is off. Non-volatile memory, such as ROM, retains its contents even when the power is off.

    To fulfill a function in an embedded environment, an Embedded System incorporates a robust motherboard inside an industrial enclosure, together with accompanying I/O (Input and Output) and Embedded OS Software. An Embedded Motherboard, which is the feature-rich tiny form factor core of an Embedded System, is the computing element. For the system to work, specialized I/O such as video capture, communication ports, and digital and analog signals are connected to the computer heart.
